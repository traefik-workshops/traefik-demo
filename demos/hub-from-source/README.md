# demos/hub-from-source

Run **Traefik Hub built from your local source** on a throwaway k3d cluster, in
front of a `whoami` workload — the dev loop for testing a Hub change on a real
cluster before it ships.

It's the same shape as [`../single-cluster`](../single-cluster) (one cluster +
Hub + whoami) with one twist: the Hub image is swapped for a locally-built one
through the `traefik/k8s` module's `custom_image_registry/repository/tag`
override. Nothing about the library changes — the demo just points it at a
different image.

## What it proves

- The library can run an arbitrary Hub image (released, preview, or from-source)
  via `custom_image_*` — no fork of the module required.
- A `build → push → apply → verify` loop that lands a source change on a cluster
  and confirms the running pod is actually your build.

## Prerequisites

- `terraform`, `k3d`, `kubectl`, `docker`
- A **Traefik Hub token** (offline JWT) — Hub is licensed, so even a from-source
  build needs one. Get it at <https://hub.traefik.io>, put it in `terraform.tfvars`.
- For the from-source path only: a **traefik-hub source checkout**. It's private
  and intentionally **not** bundled here — point `TRAEFIK_HUB_SRC` at your copy
  (default `../../../traefik-hub`). Adjust `HUB_BUILD_TARGET` to your checkout's
  image build target if it isn't `dev-image`.

## Run it

```bash
cp terraform.tfvars.example terraform.tfvars   # add your Hub token

make up         # released Hub image — proves the wiring with zero source build
make scenarios  # whoami 200 + dashboard + prints the running image

make up-dev     # build Hub from source -> push to local registry -> re-apply
EXPECT_DEV=true make scenarios   # now asserts the running image is your :dev build

make down        # destroy the cluster + delete the local registry
```

`make up-dev` switches the cluster to attach a local k3d registry, so toggling
from `make up` recreates the k3d cluster (fast, ~20s) — expected.

## How the from-source wiring works

1. `make registry` creates a k3d-managed registry at `localhost:5001`.
2. `make build-hub` runs your Hub checkout's image build (`DOCKER_REGISTRY=localhost:5001/traefik`)
   and `docker push`es `localhost:5001/traefik/traefik-hub:dev`.
3. `terraform apply -var=local_traefik_hub=true` sets `custom_image_registry=localhost:5001`,
   `custom_image_repository=traefik/traefik-hub`, `custom_image_tag=dev`. A
   `registries.yaml` mirror makes the in-cluster `localhost:5001` reference
   resolve to the registry.

For multi-cluster from-source loops, see the registry-mirror pattern in the
`k3d-unified-ingress` demo this was modelled on.

## Conventions

- Module sources are **relative** (`../../terraform/...`) so the demo always
  runs against the library in this checkout — `make validate` works offline and
  `make up` tracks your local changes. (Standalone PoCs generated by the
  `build-poc` skill pin a version instead.)
- **No secrets in git.** Real values go in `terraform.tfvars` (gitignored); only
  `terraform.tfvars.example` is committed.
